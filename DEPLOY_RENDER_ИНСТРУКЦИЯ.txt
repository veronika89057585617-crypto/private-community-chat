ИНСТРУКЦИЯ: КАК РАЗМЕСТИТЬ САЙТ, ЧТОБЫ УЧАСТНИКИ ЗАХОДИЛИ ПО ССЫЛКАМ ИЗ TELEGRAM

Главная мысль:
- Node.js нужен только серверу/хостингу.
- Участникам Node.js НЕ нужен.
- Участник получает ссылку вида https://ваш-сайт.onrender.com/join/...
- Он нажимает ссылку в Telegram и сразу попадает в нужный чат.

РЕКОМЕНДУЕМЫЙ ВАРИАНТ: Render.com

ЧТО ПОНАДОБИТСЯ:
1. Аккаунт GitHub.
2. Аккаунт Render.com.
3. Этот проект загрузить в GitHub.
4. В Render создать Web Service из этого GitHub-репозитория.
5. Обязательно подключить persistent disk, чтобы чаты, ссылки и нарушения сохранялись.

НАСТРОЙКИ RENDER:
- Runtime: Node
- Build Command: npm install
- Start Command: npm start
- Health Check Path: /api/health
- Environment variable:
  DATA_DIR=/var/data
- Disk:
  Mount Path: /var/data
  Size: минимальный доступный

ПОСЛЕ ЗАПУСКА:
Render даст публичный адрес сайта, например:
https://private-community-chat.onrender.com

Админская ссылка будет видна в Logs сервиса:
Admin link: https://.../admin/...

ЭТУ АДМИНСКУЮ ССЫЛКУ НЕЛЬЗЯ ПУБЛИКОВАТЬ.
Храните её только у администраторов.

ДАЛЬШЕ:
1. Открываете админскую ссылку.
2. Выбираете чат.
3. Создаёте ссылку участника.
4. Копируете ссылку.
5. Отправляете участнику в Telegram.
6. Участник просто кликает по ссылке — ничего не устанавливает.
